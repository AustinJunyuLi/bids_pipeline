# Requirements: v2.1 V2 Default Cutover + Legacy Archive

**Defined:** 2026-03-31
**Core Value:** Make the repo unambiguously v2-first so future reruns and
reconciliations use the observation-graph workflow without being confused by
retired v1 artifacts.

## v2.1 Requirements

### Live Surface

- [x] **CUTOVER-01**: `/deal-agent` is the live default orchestration skill and documents a clean rerun through `db-export-v2`
- [x] **CUTOVER-02**: `/reconcile-alex` is the live default post-export benchmark skill for v2 outputs
- [x] **CUTOVER-03**: `CLAUDE.md`, `docs/design.md`, and live planning docs present v2 as the default workflow

### Legacy Preservation

- [x] **LEGACY-01**: explicit legacy skills exist for the retired v1 orchestration and reconciliation flows
- [x] **LEGACY-02**: legacy-only extraction, verification, and enrichment skills are clearly marked as retired v1 surfaces
- [x] **LEGACY-03**: current v1 skill outputs are archived under `data/legacy/v1/`

### Live Data Hygiene

- [x] **DATA-01**: the pre-cutover database is preserved as `data/legacy/v1/pipeline_precutover.duckdb`
- [x] **DATA-02**: the live `data/pipeline.duckdb` file is rebuilt from v2 artifacts only
- [x] **DATA-03**: the live `data/skill/<slug>/` surface no longer stores the archived v1 directories

### Repo Hygiene

- [x] **HYGIENE-01**: skill mirrors are synced after canonical `.claude/skills/` changes
- [x] **HYGIENE-02**: tests enforce the live v2 benchmark boundary and the legacy archive split
- [x] **HYGIENE-03**: milestone-era v2.0 roadmap and requirements are archived under `.planning/milestones/`

## Out of Scope

- new extraction logic beyond the shipped v2 contract
- new benchmark policy changes beyond the live/legacy split
- UI or product-layer work
- removal of legacy runtime code paths from `skill_pipeline/`
