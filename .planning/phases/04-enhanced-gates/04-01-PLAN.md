```python
def _cycle_ranges(events: list[SkillEventRecord]) -> list[tuple[int, int]]:
    """Return inclusive index ranges for restart-delimited cycles."""
    if not events:
        return []
    ranges: list[tuple[int, int]] = []
    start_idx = 0
    for idx, evt in enumerate(events):
        if evt.event_type == "restarted" and idx > 0:
            ranges.append((start_idx, idx - 1))
            start_idx = idx
    ranges.append((start_idx, len(events) - 1))
    return ranges
```

**GATE-01: _gate_temporal_consistency(artifacts, blocks):**
1. Build block index: `{block_id: block}` from blocks
2. For each event, find its evidence blocks. In canonical mode, use span_index -> span.block_ids. In quote_first mode, use quote.block_id from the raw artifact's quotes array (via quote_ids on the event).
3. For each event with a parseable date (sort_date or normalized_hint), compare against its blocks' date_mentions where `normalized` is not None and parseable as ISO date.
4. If event date differs from ALL of its block date_mentions by >2 years (730 days): emit blocker finding with gate_id="temporal_consistency", rule_id="date_block_mismatch".
5. If fewer than MIN_DATABLE_BLOCKS blocks have parseable dates: skip entirely, no findings.
6. Return list[GateFinding].

**GATE-02: _gate_cross_event_logic(sorted_events, cycle_ranges, undated_event_ids):**
Iterate over each cycle range. Within each cycle, apply 4 rules:

Rule 1 (proposal_after_executed): Track whether an executed event has been seen. If a proposal appears after executed, emit blocker.
Rule 2 (nda_after_drop): Track set of dropped actor_ids in cycle. If nda event's actor_ids intersect dropped set and no restarted has cleared it, emit blocker.
Rule 3 (announcement_before_deadline): If final_round appears without prior final_round_ann in the same cycle, emit blocker. Same for final_round_inf/final_round_inf_ann and final_round_ext/final_round_ext_ann pairs.
Rule 4 (executed_last_in_cycle): If a SUBSTANTIVE event (not restarted) appears after executed in the same cycle, emit warning.

For undated events that appear in the sorted tail: emit a warning-level finding with rule_id="undated_event_in_sequence" if the event_type is substantive.

Per D-06: Start with these 5-7 rules. Per D-07: No rule suppression mechanism.

**GATE-03: _gate_actor_lifecycle(actors, events):**
1. Build set of NDA signer actor_ids: for each event with event_type=="nda", add its actor_ids.
2. Build set of downstream actor_ids: for each event with event_type in {"proposal", "drop", "executed", "terminated"}, add its actor_ids.
3. For each NDA signer not in downstream set: emit warning finding with gate_id="actor_lifecycle", rule_id="nda_signer_no_downstream".
4. Per RESEARCH note on Rule 5 overlap with GATE-02: implement once here in GATE-03.

**GATE-04: _gate_attention_decay(verification_findings, blocks):**
Per D-08, D-09: diagnostic only, never a blocker.
1. Build block ordinal index: `{block_id: ordinal}` from blocks.
2. For each verification finding, map block_ids to ordinals.
3. If no failure ordinals: return GateAttentionDecay(quartile_counts=[0,0,0,0], hot_spots=[], decay_score=1.0, note=None).
4. Bin failure ordinals into 4 quartiles by block position (q_size = total_blocks / 4).
5. Compute decay_score using normalized Shannon entropy: entropy / log(4). 1.0 = uniform. 0.0 = all in one quartile.
6. Find hot_spots: sliding window of 5 blocks; if 3+ failures in window, record {"block_ordinal_start": start, "block_ordinal_end": end, "failure_count": count}.
7. Add diagnostic note: if decay_score < 0.5: "Attention decay pattern detected: failures concentrated in Q{max_quartile}". If 1-2 total failures: "Too few failures for statistical significance".

**Entry point: run_gates(deal_slug, *, project_root):**
1. Build paths, ensure output directories.
2. Load extract artifacts via load_extract_artifacts(paths).
3. Load chronology blocks via _load_chronology_blocks(paths).
4. Run GATE-01, GATE-02, GATE-03 (collect findings).
5. For GATE-02: sort events by date first, compute cycle ranges on sorted events.
6. For GATE-04: load verification_findings.json (handle missing gracefully -- if file missing, skip attention decay with None).
7. Build GateReport with all findings + attention_decay.
8. Write report to paths.gates_report_path as JSON.
9. Return 1 if any finding has severity=="blocker", else 0.

**Quote_first vs canonical mode handling:**
- GATE-01 quote_first: map event quote_ids -> quotes -> block_id to find evidence blocks. Use DateHint.normalized_hint for event date.
- GATE-01 canonical: map event evidence_span_ids -> span_index -> span.block_ids. Use ResolvedDate.sort_date for event date.
- GATE-02, GATE-03: use _get_actor_event_records() to get the right record list regardless of mode.
- GATE-04: always uses verification_findings.json (mode-independent).
